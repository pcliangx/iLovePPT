## 决策

**最优 hybrid 权重:`text_weight=0.8, image_weight=0.2`**

决策规则(按顺序):

1. hit_rate 必须 100%(7/7)· 任何 query miss 直接 disqualify
2. **排除纯模式**(`(1.0,0.0)` 等价 `--mode text`,`(0.0,1.0)` 等价 `--mode image`)· hybrid default 必须真的混合,否则 hybrid mode 沦为冗余
3. 在满足 #1 + #2 的混合组里,选 `avg_gap` 最大(top1 跟 top2 区分度高)
4. tie-break:`low_gap_count` 少的优先(更少边缘 query)

该组指标:`hit_rate=7/7 · avg_top1=0.6328 · avg_gap=0.1059 · low_gap=1/7`

### 数据观察 · 为什么纯 text 数字最好却不当 hybrid default

1. **现象**:6 组里 `(1.0, 0.0)` 的 avg_top1=0.7532 / avg_gap=0.1229 全面最优,image 权重越大 avg_top1 越线性下降,(0.0, 1.0) 跌到 0.1517 还 miss 1 query。
2. **解释**:tongyi-embedding-vision-plus 把 text 和 image 编到同一空间,但 image embedding 之间的余弦距离系统性比 text-text 大 · 导致 `(1 - image_dist)` 项普遍很小,稀释了 text 信号的强度。
3. **后果**:如果把 hybrid default 设成 `(1.0, 0.0)`,则 `--mode hybrid` 退化成 `--mode text`,用户喊 `hybrid` 等于白喊。
4. **结论**:在 7 query golden set 上,**最优 hybrid 混合**是 `(0.8, 0.2)` · text 主导(避开 image 信号弱)+ 20% image 弱掺杂(保留视觉差异化能力)· hit_rate 仍 100%,avg_gap 0.1059(仅次于纯 text),low_gap 1/7。

### Per-query 异常 · 视觉密集 query 的潜力

`斜切条纹 几何工业` 是唯一一个 image 权重越大 gap 越大的 query — 从 (1.0,0.0) 的 0.0376 升到 (0.0,1.0) 的 0.0760。说明:

- 当 query 是纯视觉描述(无业务语义)时,image embedding 反而更能区分
- 当前 golden set 7 query 中仅 1 个是视觉 query,故 average 被语义 query 主导
- 若后续扩 query 集到含更多视觉 query,可能 hybrid 最优值会偏向 image 一侧

### Action

- 已回写 `library/search.py` 的 `text_weight=0.8 / image_weight=0.2` default(`search()` + CLI argparse)
- `--mode text` 行为不变(等价 `(1.0, 0.0)`)
- `--mode image` 行为不变(等价 `(0.0, 1.0)`)
- `--mode hybrid` 默认 `(0.8, 0.2)`,需要更视觉时手动 `--text-weight 0.4 --image-weight 0.6`
