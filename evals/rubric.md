# iLovePPT Eval 评分标准（PPTEval 三维）

参照 PPTAgent 的 PPTEval 方法,沿三个维度评 deck：**Content（内容）/ Design（设计）/ Coherence（连贯）**。

## 用法

- 评一份**真实生成的 deck**：三维全评。
- 跑**固定 eval 集回归**（`run_eval.sh`）：固定夹具是 filler 内容,Content / Coherence 不适用 —— **Design 维是固定夹具的确定性回归信号**;Content / Coherence 仅在评真实 deck 时打分。

## 一、Content（内容）—— 每页信息质量

- **C1** 标题是行动式标题（完整结论句),不是话题标签
- **C2** 文案具体：数字 > 形容词,无空泛词（"高效" / "创新" / "领先"）
- **C3** 句式一致（bullet 全动宾 或 全名词性,不混用）
- **C4** 字数在 content-writing.md 各 layout 上限内
- **C5** 每页内容真正支撑该页意图,无凑数
- **C6** 图文互补：`pic_text` 页的图与右侧卡片说明一致,不是图文各说各的

## 二、Design（设计）—— 视觉质量【固定夹具回归主信号】

- **D1** 无元素重叠
- **D2** 无文字溢出 / 截断
- **D3** 中文字体未 fallback（不是 Arial / 花体）
- **D4** 标题与内容区间距合理
- **D5** 颜色对比度足够（不深底深字 / 浅底浅字）
- **D6** layout 与意图相符
- **D7** 数字 / 图无 margin 偏移
- **D8** 配色在 BRAND_* / GRAY_* 套色板内
- **D9** 留白边界达标（离页边 ≥ 0.5"）
- **D10** 内容区无大片异常空白（短内容应居中 / 均布）
- **D11** 列 / 行对齐网格
- **D12** 表格无意外 banding
- **D13** 无 emoji 误用 / 方块
- **D14** 装饰大字号未异常换行

## 三、Coherence（连贯）—— 整份 deck 的逻辑流

- **H1** Ghost deck test：所有 action title 顺序读下来,独立成一条完整故事线
- **H2** 论证结构成立：背景 → 问题 → 方案 → 结论（或金字塔:结论先行 + 论据支撑）
- **H3** 章节扉页与 toc、与内容页一一对应
- **H4** summary 给出结论,不是重列 toc 章节名
- **H5** 页序节奏合理（封面 → 目录 → 章节 → 内容 → 总结 → 封底）
- **H6** 图密度合理（每 4-5 页 ≥ 1 图,若内容适合配图）

## 打分

每维记录 fail 的项号。一个 deck 的某维得分 = 该维检查数 − 该维 fail 数。
回归判定：新 scorecard 对比 `baseline/scorecard.md`,**某一维 fail 项变多 = 该维回归**。
